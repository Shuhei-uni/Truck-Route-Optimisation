# Woolworths Truck Routing – Project Overview

## Quick Summary
This repository contains a complete end‑to‑end solution for optimizing Woolworths truck routing across 64 stores. It builds a **global linear program (LP)**, simulates demand, schedules shifts, and visualises the results in a **premium interactive web report**.

---

## What’s Inside
| Component | Description |
|-----------|-------------|
| `src/` (or the root `Project code/`) | Python scripts that generate the LP, run the simulation, perform fleet‑size analysis, and create the data bundle for the web app. |
| `Interactive_Report/` | Vanilla HTML/JS/CSS dashboard with dark‑glassmorphism styling, Leaflet map, Chart.js charts, and the new **Live Simulation** sidebar tab. |
| `data/` | CSV files (`WoolworthsLocations.csv`, demand data, route lists) used by the scripts. |
| `visualisations_gallery.md` (artifact) | Sample plots generated by `Visualisations.py`. |

---

## Core Algorithms & Technical Details
- **LP Formulation (`Global_LP.py`)**
  - **Variables**: `WW_routes[i]` (binary, route served by Woolworths truck) and `outsourced_routes[i]` (binary, route outsourced to Mainfreight).
  - **Objective**: Minimise total cost = owned‑truck operating cost + overtime + Mainfreight block cost + annual maintenance.
  - **Constraints**:
    1. Every store is covered exactly once (WW or Mainfreight).
    2. A route cannot be both WW and outsourced.
    3. Global fleet cap: `sum(WW_routes) <= total_fleet_size * 2` (two shifts per truck).
  - Solved with **PuLP/GLPK** on the Python backend. The Live Simulation triggers the backend pipeline to regenerate and re-render the dashboard.

- **Shift Scheduler (`Shift_Scheduler.py`)** – First‑Fit‑Decreasing (FFD) algorithm packs routes into two 4‑hour shift slots per truck.

- **Fleet‑Size Sweep (`Fleet_Size_Analysis.py`)** – Re‑runs the LP for fleet sizes 10‑24, adds truck‑ownership cost, and plots the total annual cost curve.

- **Simulation (`Simulation_Fixed_Schedule.py`)** – Uses 100 days of stochastic demand, applies overtime rules, and triggers Mainfreight overflow trips when a route exceeds 20 pallets.

## Getting Started (Windows)
1. **Clone / copy the repository** to a convenient location.
2. **Create a virtual environment**
   ```powershell
   cd "Project code"
   python -m venv .venv
   .\.venv\Scripts\Activate.ps1   # or Activate.bat for cmd
   ```
3. **Install dependencies** (the required packages are listed in `requirements.txt` – if the file is missing, run the commands below):
   ```powershell
   pip install pulp pandas matplotlib folium glpk pyglpk
   ```
4. **Generate the data bundle for the web app**
   ```powershell
   python Generate_Report_Data.py   # creates data.js and params.json
   ```
5. **Start the web server** (must use the custom API server):
   ```powershell
   python run_server.py
   ```
6. Open a browser and navigate to `http://localhost:8080` (or the port shown in the terminal).
   - The **Live Simulation** tab lets you tweak fleet size, costs, overtime rate, overflow penalty, and demand distribution. Changes will trigger the Python backend API (`Globel_LP.py`, `Shift_Scheduler.py`, etc.) and instantly refresh the entire dashboard report!
